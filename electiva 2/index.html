    • ¿Qué es Git?

    Es un sistema de control de versiones distribuido que permite a los desarrolladores llevar un registro de los cambios en sus archivos y trabajar en equipo de manera eficiente.
    Es un software de código abierto que se utiliza para gestionar el historial de un proyecto, permitiendo revertir a versiones anteriores, crear ramas para experimentos y fusionar cambios de diferentes desarrolladores.

    Características

  • Distribuido y Descentralizado:

    • A diferencia de los sistemas de control de versiones centralizados, en los que se almacena una única copia del repositorio, Git es distribuido y como desarrolladores tenemos una copia completa del historial de cambios.Esto nos permite trabajar de forma independiente y fusionar nuestros cambios en el repositorio principal de manera eficiente.

  • Rastreo Preciso de Cambios:

    • Git realiza un seguimiento preciso de los cambios realizados en cada archivo a lo largo del tiempo. Esto nos permite como desarrollador ver el historial completo de cambios, incluidas las diferencias entre versiones y la posibilidad de regresar a versiones anteriores en caso de problemas.

  • Ramificación Eficiente:

    • Git hace que la creación y gestión de ramas branches sea sencillo. Como desarrollador podemos crear ramas para trabajar con nuevas características o arreglos sin afectar la rama principal. Esto fomenta el desarrollo paralelo y facilita la colaboración en equipos grandes.

  • Fusiones Simplificadas:

    • La capacidad de fusionar cambios entre diferentes ramas es una característica clave de Git. Como desarrollador podemos combinar cambios de manera eficiente, lo que facilita la integración continua y evita conflictos de código.

  • Soporte para proyectos de cualquier tamaño:

    • Git se puede utilizar en proyectos de cualquier tamaño, desde proyectos pequeños con un solo desarrollador hasta proyectos grandes y complejos con cientos de desarrolladores trabajando en paralelo.

    Elementos

  • Repositorio Git:

    • Es el lugar donde se almacenan todos los archivos y la historia de cambios de un proyecto.

  • Commit:

    • Es un registro de cambios en el repositorio, que contiene información sobre los archivos modificados y el autor del cambio.

  • Branches:

    • Son ramas de desarrollo separadas donde se pueden trabajar en paralelo sin afectar la rama principal.

  • Merge:

    • Es el proceso de combinar cambios de una rama en otra.

  • Clone:

    • Es hacer una copia de un repositorio remoto en tu máquina local.

  • Pull:

    • Es obtener los cambios de un repositorio remoto y fusionarlos con tu repositorio local.

  • Push:

    • Es enviar tus cambios locales a un repositorio remoto.

  • Fork:

    • Es hacer una copia de un repositorio en tu cuenta de GitHub para poder trabajar en ello sin afectar al repositorio original.

  • Pull request:

    • Es una solicitud para que los cambios que has hecho en tu rama sean fusionados en una rama principal.

  • Tag:

    • Es una forma de marcar un punto específico en la historia de un repositorio para indicar una versión específica del software.

    Comandos basicos de Git

    Iniciar repositorio

  • Iniciar un repositorio Git en la carpeta donde esta el proyecto:

    • git init

  • Clonar el repositorio de github o bitbucket:

    • git clone

  • Añadir todos los archivos para el commit:

    • git add .

  • Hacer el primer commit:

    • git commit -m "Texto que identifique por que se hizo el commit"

  • Subir al repositorio:

    • git push origin master

    Git Clone

  • Clonar el repositorio de github o bitbucket:

    • git clone <'url'>

    Git add

  • Añadir todos los archivos dentro de un directorio y de una extensión especifica:

    • git add docs/*.txt

  • Añadir todos los archivos dentro de un directorio:

    • git add docs/

    Git push

  • Subir al repositorio:

    • git push <'origien'> <'branch'>

  • Subir un tag:

    • git push --tags

    Git diff

  • Mostrar los cambios realizados a un archivo:

    • git diff
    git diff --staged

    ¿Qué es Github?

    Es una plataforma de alojamiento web y una comunidad donde los desarrolladores pueden almacenar, compartir, y colaborar en proyectos de software utilizando el sistema de control de versiones Git.


    Características

    1. Repositorios:

      2. Permite crear y alojar repositorios de proyectos de software. Los repositorios pueden ser públicos o privados.

    2. Control de versiones:

      3. Utiliza el sistema de control de versiones Git, lo que permite a los usuarios llevar un registro de los cambios realizados en el código de un proyecto.

    3. Colaboración:

      4. Facilita la colaboración entre diferentes desarrolladores, ya que permite a los usuarios trabajar juntos en un mismo proyecto, realizar comentarios, revisar cambios y fusionar ramas.

    4. Seguridad:

      5. Ofrece herramientas de seguridad como autenticación de dos factores, cifrado de datos y revisiones de código para

    5. Integraciones:

      6. Se integra con una amplia variedad de herramientas y servicios de desarrollo de software, lo que facilita la automatización de tareas y la integración de diferentes herramientas en el flujo de trabajo.

    Elementos

  • Repositorios:

    • Son espacios donde se almacenan los archivos y código de un proyecto. Cada repositorio tiene su propio historial de cambios, ramas y colaboradores.

  • Commits:

    • Son las actualizaciones o cambios realizados en los archivos dentro de un repositorio. Cada commit registra quién hizo el cambio, cuándo y qué se modificó.

  • Ramas:

    • Son versiones paralelas del código dentro de un repositorio. Se utilizan para desarrollar nuevas funcionalidades, corregir errores o trabajar en diferentes aspectos de un proyecto sin afectar la versión principal.

  • Solicitudes de extracción (Pull Requests):

    • Permite a los colaboradores proponer cambios al código de un repositorio y solicitar que se incorporen a la versión principal.

  • Problemas (Issues):

    • Son una forma de realizar un seguimiento de tareas, bugs o mejoras en un proyecto. Los problemas se pueden asignar a colaboradores, etiquetar, comentar y cerrar una vez resueltos.

  • Acciones (Actions):

    • Son tareas automatizadas que se pueden configurar para ejecutarse en respuesta a eventos específicos en un repositorio, como la creación de un commit o una solicitud de extracción.

    ¿Cómo funcionan Git y GitHub de manera conjunta?

    Al cargar archivos en GitHub, los almacenarás en un "repositorio de Git". Esto significa que al realizar cambios (o "confirmaciones") en los archivos de GitHub, Git se iniciará automáticamente para realizar el seguimiento de los cambios y administrarlos.

    Hay muchas acciones relacionadas con Git que puedes completar en GitHub directamente en el navegador, como crear un repositorio de Git, crear ramas y cargar y editar archivos.

    Pero la mayoría de los usuarios trabajan en sus archivos localmente (en su propio ordenador), luego sincronizan continuamente estos cambios locales y todos los datos de Git relacionados, con el repositorio central "remoto" en GitHub. Hay muchas herramientas que puedes usar para hacerlo, como GitHub Desktop. Cuando empieces a colaborar con otros y todos necesitéis trabajar en el mismo repositorio al mismo tiempo, haréis lo siguiente continuamente:

  • Extraer todos los cambios más recientes realizados por los colaboradores del repositorio remoto en GitHub.

  • Insertar tus propios cambios en el mismo repositorio remoto en GitHub.

    • Git determina cómo combinar inteligentemente este flujo de cambios y GitHub te ayuda a administrarlo mediante características como las "solicitudes de cambios".
